1. Introducción al Patrón MVC

🎯 ARQUITECTURA MVC (Model-View-Controller) separa la aplicación en tres capas:

  • Modelo: Datos y lógica de negocio (clase Datos).
  • Vista: Interfaz de usuario (archivos .cshtml con HTML/Razor).
  • Controlador: Gestiona peticiones, interactúa con el modelo y selecciona la vista.

✅ Beneficios: mantenibilidad, reutilización, escalabilidad y separación de responsabilidades.

// Datos.cs
namespace EjercicioCompleto.Models
{
    public class Datos
    {
        public string? usuario_ { get; set; }      // Nombre de usuario (identificador)

5. Controlador HomeController (código íntegro)

using EjercicioCompleto.Models;
using Microsoft.AspNetCore.Mvc;

namespace EjercicioCompleto.Controllers
{
    public class HomeController : Controller
    {
        static List<Datos> listado = new List<Datos>();

        public IActionResult Index() => View();

        public IActionResult validar(string txtUsuario, string txtContraseña)
        {
            if (txtUsuario != null && txtContraseña != null)
            {
                var traeDatos = listado.Find(l => l.usuario_ == txtUsuario);
                if (traeDatos == null)
                    ViewBag.respuesta = "Usuario ingresado no existe en el sistema";
                else if (traeDatos.contraseña_ != txtContraseña)
                    ViewBag.respuesta = "Contraseña incorrecta";
                else if (traeDatos.habilitado_ != "on")
                    ViewBag.respuesta = "Usuario no se encuentra HABILITADO";
                else
                    ViewBag.respuesta = "Bienvenido usuario: " + txtUsuario;
            }
            else
                ViewBag.respuesta = "usuario vacio";

            return View("Index");
        }

        public IActionResult Agregar()
        {
            ViewBag.respuesta = listado;
            return View();
        }

        public IActionResult Nuevo(string txtUsuarioNuevo, string txtContraseñaNuevo, string chkHabilitado)
        {
            listado.Add(new Datos()
            {
                usuario_ = txtUsuarioNuevo,
                contraseña_ = txtContraseñaNuevo,
                habilitado_ = chkHabilitado
            });
            ViewBag.respuesta = listado;
            return View("Agregar");
        }

        public IActionResult Editar(string txtEditarUsuario, string txtEditarContraseña, string chkEditarHabilitado)
        {
            ViewBag.UsuarioEditado = txtEditarUsuario;
            ViewBag.ContraseñaEditado = txtEditarContraseña;
            ViewBag.HabilitadoEditado = (chkEditarHabilitado == "on") ? "checked" : "";
            return View();
        }

        public IActionResult EditarDatos(string txtUsuarioNuevo, string txtContraseñaNuevo, string chkHabilitadoNuevo)
        {
            var editaDatos = listado.Find(l => l.usuario_ == txtUsuarioNuevo);
            if (editaDatos != null)
            {
                editaDatos.contraseña_ = txtContraseñaNuevo;
                editaDatos.habilitado_ = chkHabilitadoNuevo;
            }
            ViewBag.respuesta = listado;
            return View("Agregar");
        }

        public IActionResult Eliminar(string txtUsuarioEliminar)
        {
            ViewBag.datoUsuario = txtUsuarioEliminar;
            return View();
        }

        public IActionResult confirmaVolver()
        {
            ViewBag.respuesta = listado;
            return View("Agregar");
        }

        public IActionResult confirmaEliminar(string txtUsuarioDatoEliminar)
        {
            var eliminaDatos = listado.Find(l => l.usuario_ == txtUsuarioDatoEliminar);
            if (eliminaDatos != null)
                listado.Remove(eliminaDatos);
            ViewBag.respuesta = listado;
            return View("Agregar");
        }
    }
}

7. Flujo CRUD paso a paso

  • CREATE: Formulario Agregar → POST /Home/Nuevo → listado.Add() → redirige a Agregar.
  • READ: Vista Agregar recibe listado en ViewBag.respuesta y lo muestra en tabla.
  • UPDATE: Botón Editar envía datos ocultos → Editar.cshtml → EditarDatos() modifica.
  • DELETE: Botón Eliminar → Eliminar.cshtml confirmación → confirmaEliminar() remueve.

8. Análisis técnico y funcionamiento

🔍 CLAVE ViewBag: objeto dinámico para pasar datos entre controlador y vista. Inputs hidden: mantienen el identificador (usuario) en edición/eliminación. Checkbox: mapea a string "on"/null. Lista estática: los datos persisten solo en memoria mientras la app está activa.

9. Buenas prácticas y mejoras sugeridas

  • ✅ Seguridad: Hash de contraseñas con BCrypt.
  • ✅ Persistencia: Reemplazar lista estática por Entity Framework + SQL Server.
  • ✅ Identificador: Usar Id autoincremental en lugar del nombre de usuario.
  • ✅ Validaciones: DataAnnotations ([Required], [StringLength]).
  • ✅ Manejo de errores: Try-catch y mensajes amigables.
